标签: JavaScript设计模式

  1. JS设计模式——装饰者模式

    2016-04-25

    给对象动态地增加职责的方式称作装饰者模式，它是在程序运行期间给对象动态添加职责。

  2. JS设计模式——发布-订阅模式

    2016-04-23

    发布-订阅模式又叫观察者模式，它定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到通知。

  3. JS设计模式——单例模式

    2016-04-21

    保证一个类只有一个实例，并提供一个全局的访问点。

  4. JS设计模式——策略模式

    2016-04-17

    定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换。实际中“算法”的含义可以扩散开来：算法也可以指代不同的“业务规则”。
